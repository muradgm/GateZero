### **2. 3D Asset Requirements**

#### **A. Environment: Evidence Gate Interface**
- **Purpose:** A clean, modular, and interactive 3D interface that represents the core of TraderFrame's decision-making process.
- **Design Principles:**
  - **Calm precision:** Minimalist, uncluttered, and highly organized.
  - **Transparency:** Visual cues for filtering, approval, and recording.
  - **User control:** Interactive elements that allow users to manipulate data and evidence.
- **Key Components:**
  - **Evidence Panels:** Modular, scrollable, and collapsible.
  - **Filtering UI:** Sliders, toggles, and search bars.
  - **Approval Workflow:** Visual indicators (e.g., green/red status lights, progress bars).
  - **Recording Interface:** Timeline, annotations, and version control.
  - **Risk Indicators:** Color-coded risk levels, icons, and alerts.
- **Style:** Modern, neutral tones with subtle gradients and soft shadows. Use of glassmorphism for transparency and clarity.

#### **B. 3D Assets: Interactive Elements**
- **Evidence Panels (3D Models):**
  - **Format:** Modular UI panels with interactive elements.
  - **Materials:** Glass-like, semi-transparent, with subtle glow.
  - **Interactivity:** Clickable, draggable, collapsible.
- **Filtering UI (3D Models):**
  - **Format:** Sliders, toggles, and search bars.
  - **Materials:** Sleek, metallic, with soft highlights.
  - **Interactivity:** Drag-to-filter, toggle states, search input.
- **Approval Workflow (3D Models):**
  - **Format:** Status indicators, progress bars, and confirmation buttons.
  - **Materials:** Monochrome with glowing accents.
  - **Interactivity:** Click-to-approve, drag-to-verify.
- **Recording Interface (3D Models):**
  - **Format:** Timeline, annotations, version control.
  - **Materials:** Clean, white, with subtle color coding.
  - **Interactivity:** Drag-to-annotate, click-to-version.
- **Risk Indicators (3D Models):**
  - **Format:** Color-coded icons and alerts.
  - **Materials:** High-contrast, with dynamic lighting.
  - **Interactivity:** Hover-to-expand, click-to-details.

#### **C. Environment: Data Visualization Layer**
- **Purpose:** Visual representation of market data and evidence flows.
- **Design Principles:**
  - **Transparency:** Data flows are visualized with clear, unobstructed paths.
  - **Calm precision:** Data is presented in a structured, readable format.
- **Key Components:**
  - **Data Flow Diagrams:** Animated, interactive flowcharts.
  - **Market Indicators:** Real-time data visualization (e.g., price charts, volume).
  - **Evidence Tracing:** Visual lineage of data sources and approvals.
- **Style:** Clean, minimal, with dynamic lighting and subtle animations.

#### **B. Asset Specifications**
- **Resolution:** 4K (3840 x 2160)
- **Format:** FBX, glTF, USDZ (as needed)
- **Material Format:** PBR (Physically Based Rendering)
- **Lighting:** HDR environment maps with soft shadows
- **Animation:** Subtle, smooth transitions for interactivity
- **Export Settings:** Include UV maps, material IDs, and animation curves

#### **C. Asset Pipeline**
- **Modeling:** Blender, Maya, or 3ds Max
- **Texturing:** Substance Painter, Photoshop
- **Rigging & Animation:** Maya or Blender (for UI elements)
- **Export:** FBX with animation, materials, and textures
- **Integration:** Unity or Unreal Engine for final rendering and interactivity
