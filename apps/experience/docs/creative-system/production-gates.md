# Production Gates

## Gate C0 — Product truth

Required:

- Product definition approved
- Wedge approved
- User action model defined
- Claims and prohibited language documented

## Gate C1 — Creative direction

Required:

- Three genuinely different territories reviewed
- One north-star concept selected
- Memorability logic stated
- Concept is recognizably TraderFrame-specific

## Gate C2 — Information model

Required:

- Evidence states defined
- Probability and confidence distinguished
- Contradiction visible
- Risk can override recommendation
- Invalidation state defined

## Gate C3 — Experience narrative

Required:

- Beat sheet approved
- Interaction moments approved
- Camera intent approved
- Reduced-motion narrative approved

## Gate C4 — Art direction

Required:

- Composition studies
- Material and lighting board
- Type and UI hierarchy
- Colour semantics
- Desktop and mobile keyframes

## Gate P1 — Concept art lock

Required:

- Hero view
- Motion-state views
- Detail views
- Material references
- Feasibility review

## Gate P2 — Animatic lock

Required:

- Full sequence
- Timing map
- Camera paths
- UI transition
- Sound direction

## Gate P3 — Asset and design-system lock

Required:

- UI tokens and components
- Asset manifest
- Geometry and texture budgets
- Naming and export conventions

## Gate P4 — Feature complete

Required:

- All narrative states implemented
- Interaction complete
- Product wording validated
- Responsive and reduced-motion modes complete

## Gate P5 — Release candidate

Required:

- Typecheck and build pass
- Cross-browser QA
- Accessibility review
- Performance budgets pass
- WebGL fallback
- Analytics and error monitoring
- Deployment and rollback documentation
