# Conviction Atlas — Visual Operating System v1.0

This document is the visual source of truth for the TraderFrame immersive experience.

It governs concept art, storyboards, 3D assets, shaders, motion, interface transitions, and implementation reviews.

## 1. Intelligence language

The system visualizes interpreted meaning rather than generic data.

### Primitive

The base visual primitive is the **living intelligence thread**.

Every thread may encode:

- source,
- category,
- age,
- confidence,
- importance,
- agreement,
- contradiction.

### Behaviours

| Meaning | Behaviour |
|---|---|
| Agreement | Threads converge and merge |
| Conflict | Threads split, pull apart, or form tension |
| Expiry | Thread loses brightness, thickness, and influence |
| Importance | Thread becomes thicker and more persistent |
| Correlation | Threads form shared flow and contour direction |
| Noise | Thread remains unstable and fails to shape the terrain |

### Rejected motifs

- floating cubes,
- random particles,
- decorative dots,
- literal news cards flying through space,
- generic AI neural networks.

## 2. Geometry language

Geometry should appear to be generated by market forces rather than constructed as a machine.

### Primary structures

- topographic contour fields,
- vector flows,
- fluid surfaces,
- geological folds,
- pressure ridges,
- valleys,
- fractures,
- route networks.

### Formal rule

Hard surfaces may appear only when the landscape begins transforming into the product interface.

## 3. Material language

### Primary materials

| Material | Role |
|---|---|
| Obsidian | Base terrain and depth |
| Smoked glass | Uncertainty and layered context |
| Polished graphite | Stable structure and product surfaces |
| Optical acrylic | Evidence lenses and selected paths |
| Liquid light | Intelligence flow and conviction |

### Material restrictions

- Avoid excessive chrome.
- Avoid glossy sci-fi machinery.
- Avoid strong bloom that destroys contour detail.
- Preserve black levels and readable silhouettes.

## 4. Motion language

Motion is semantic, not decorative.

### Master motion sequence

```text
accelerate
→ branch
→ collide
→ slow
→ stabilize
→ resolve
```

### State behaviour

| State | Motion |
|---|---|
| Raw signal | Fast, directional, unstable |
| Correlated signal | Aligned and increasingly coherent |
| Supporting evidence | Pulls terrain upward and strengthens routes |
| Contradiction | Applies lateral tension and fractures paths |
| Risk | Slows motion and deforms or collapses terrain |
| Conviction | Becomes calm, stable, and persistent |
| Decision | Geometry aligns and converts into interface structure |

### Forbidden motion

- endless rotation,
- ambient floating without meaning,
- pulsing every object,
- abrupt recommendation flashes,
- camera movement that does not reveal information.

## 5. Landscape language

The landscape is the reasoning engine made visible.

| Variable | Landscape effect |
|---|---|
| Probability | Elevation and route strength |
| Confidence | Clarity and persistence |
| Contradiction | Fracture and branching |
| Risk | Valley, heat, instability, collapse |
| Liquidity | Flow continuity |
| Volatility | Surface turbulence |
| Time decay | Erosion and fading |

The field should never look like arbitrary fantasy terrain. Every deformation must map to a product variable.

## 6. Decision language

Conviction is gradual.

1. Many routes appear.
2. Evidence changes their relative strength.
3. Contradictions weaken or divide them.
4. Risk deforms or removes unsafe options.
5. One route becomes the strongest current path.
6. Alternatives and invalidation remain visible.

The recommended route must feel conditional and revisable.

## 7. Interface language

The product UI emerges from the same world.

### Transformation grammar

```text
contours flatten
→ ridges align
→ surfaces become panels
→ routes become charts and evidence rows
→ terrain labels become interface metadata
→ TraderFrame workspace resolves
```

### Rule

No cut, fade-to-dashboard, or unrelated overlay should be used as the primary transition.

## 8. Camera language

| Stage | Camera behaviour |
|---|---|
| Observe | High, distant, patient |
| Correlate | Slow lateral drift reveals relationships |
| Form Thesis | Descend as terrain rises |
| Challenge | Hold or slow while fractures develop |
| Conviction | Follow the strongest route with restrained acceleration |
| Execute | Dive and align with the interface plane |

The camera behaves like an analyst discovering structure, not a drone performing spectacle.

## 9. Colour semantics

| Colour | Meaning |
|---|---|
| Cyan | Verified evidence and active intelligence |
| Violet | Selected thesis and conviction |
| Amber | Uncertainty, counter-thesis, and review |
| Red | Invalidation and prohibited risk only |
| Silver | Baseline context and inactive routes |

Colour must never be assigned solely for decoration.

## 10. Review test

Every visual or motion proposal must answer:

> What does this tell the trader about the quality of this decision?

If the answer is unclear, the element should be redesigned or removed.
