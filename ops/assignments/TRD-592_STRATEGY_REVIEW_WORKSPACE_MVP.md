# TRD-592 Strategy Review Workspace MVP

Status: accepted

## Goal

Create the first usable operator-facing research workflow: one local research case visible
end-to-end.

## Scope

- Strategy idea.
- Data snapshot.
- Backtest evidence.
- Metric report.
- Risk review.
- Operator note.
- Outcome log.
- Learning event.
- Simulation evidence detail.
- Artifact inventory.
- Blocked-scope reminder.

## Blocked Scope

- Action controls, account connectivity, live routes, automated decisions, final recommendations,
  output channels, readiness claims, approval claims, and performance claims.

## Acceptance

Accepted when the Command Center renders the read-only Strategy Review Workspace from local fixture
data and keeps limitations adjacent to evidence.
