# TRD-476 Frontend Visual Hierarchy Direction

## Goal

Set frontend visual hierarchy direction for future read-only evidence surfaces.

## Scope

- Gate: `G2_PAPER_TRADING`.
- Scope: `paper_simulation_planning_only`.
- Surface: hierarchy, density, panel prominence, risk visibility, limitation placement, and operator
  scanning.

## Blocked Scope

- No marketing hero, trading-terminal styling, oversized performance visuals, profit-first charts,
  approval badges, readiness badges, or execution-oriented controls.

## Required Outputs

- Visual hierarchy direction.
- QA/security, risk, and orchestrator review records.
- Tracker, docs index, command-center, and guard-index updates.

## Acceptance Criteria

- Risk and limitations are visually prominent.
- Evidence hierarchy supports repeated operator review.
- Next task remains implementation readiness blocker audit.
