# Gate 2 Frontend Visual Hierarchy Direction

## Summary

TRD-476 sets visual hierarchy direction for future read-only evidence surfaces.

## Direction

- First: gate, boundary, and blocked scope.
- Second: evidence quality and limitations.
- Third: risk review and blockers.
- Fourth: manual operator workflow state.
- Fifth: source links and validation records.

## UI Posture

The app should feel calm, dense, operational, and trustworthy. Avoid marketing composition,
profit-first charts, trading-terminal affordances, and decorative layouts that hide risk.

## Result

The interface should help the operator scan truth before action, with risk and limitations always
near evidence.
