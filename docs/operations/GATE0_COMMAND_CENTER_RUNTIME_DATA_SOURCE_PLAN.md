# Gate 0 Command Center Runtime Data Source Plan

## Purpose

This plan defines how the command center may later read Gate 0 operating evidence without expanding
scope.

## Allowed Future Inputs

- `ops/runtime/tracklist.md`
- `ops/runtime/progress/GATE0_PROGRESS_SNAPSHOT.md`
- Review records under `ops/runtime/reviews/`
- Local validation command output generated by Gate 0 scripts
- Local docs under `docs/operations/`

## Blocked Inputs

- Broker, exchange, or account APIs
- External execution systems
- Secret material
- External market feeds
- Strategy recommendation or prediction services
- Public deployment telemetry as a source of trading authority

## Implementation Position

The current dashboard remains static. A later data-source implementation must be local, read-only,
deterministic, and covered by Gate 0 drift guards before replacing static values.
