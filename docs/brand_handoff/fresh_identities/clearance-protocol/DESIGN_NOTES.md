# TraderFrame Fresh Identity - Clearance Protocol

## Posture

Compliance-grade, procedural, precise.

This direction treats TraderFrame as a clearance layer before any risky action. It feels closer to
an operator checklist, approval stamp, and controlled access system than a fintech logo.

## Color System

- Paper: `#F4EFE6`
- Ink: `#17202A`
- Clearance Red: `#C13B32`
- Steel: `#6B7280`
- Signal Blue: `#2F6FED`

## Typography

- Display: `IBM Plex Sans Condensed`
- UI/data: `IBM Plex Mono`
- Fallback: `Arial Narrow`, `Inter`, `Arial`

## Symbol

The symbol is a square clearance stamp with a clearer first read: a control gate, a red risk stop
line, and a blue approval check. The mark should communicate "permitted only after risk clearance"
without needing the viewer to decode abstract bars or nodes.

## Best Use

Use this if TraderFrame should feel like a serious operator protocol with institutional restraint.
